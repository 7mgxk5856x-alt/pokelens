# コードレビュー

引数(任意): ファイル/ディレクトリパス(例: `/review-code src/logic/power-index-calc.js`)

引数を省略した場合は **HEAD との差分**(`git diff HEAD`、ステージ済み + 未ステージ)をレビュー対象とする。

## 実行方法

```bash
claude
# パス指定
> /review-code src/logic/power-index-calc.js
# 引数なし: 作業ツリーの未コミット変更をレビュー
> /review-code
```

## 手順

### ステップ1: 対象の決定

- **引数あり**: 指定されたファイル/ディレクトリが存在するか確認する。存在しない場合は「対象が見つかりません。パスを確認してください」とユーザーに報告して終了する。
- **引数なし**: 以下を実行して差分の対象ファイル一覧と内容を取得する。
  ```bash
  git diff HEAD --name-only   # 変更対象ファイル一覧（ステージ済み + 未ステージ）
  git diff HEAD               # 実際の差分内容
  ```
  対象ファイルが 0 件の場合は「HEAD との差分がありません。レビュー対象がないため終了します」とユーザーに報告して終了する。

### ステップ2: code-reviewerサブエージェント起動

Task tool を使用して code-reviewer サブエージェントを起動する。プロンプトは引数の有無で切り替える。

**引数ありの場合**:
- subagent_type: "code-reviewer"
- description: "Code detailed review"
- prompt: "[対象パス]を詳細にレビューしてください。\n\n`code-reviewer` の定義にある全観点（8観点）で評価してください（観点はエージェント定義を単一の真実源とし、ここでは列挙しない）。\n\nレビューレポートを作成してください。"

**引数なしの場合**(HEAD との差分をレビュー):
- subagent_type: "code-reviewer"
- description: "Code detailed review (HEAD diff)"
- prompt: "HEAD との差分（`git diff HEAD`）を対象にコードレビューを実施してください。\n\n対象ファイル一覧:\n[git diff HEAD --name-only の結果]\n\n手順:\n1. `git diff HEAD` を実行して実際の差分を取得する\n2. 必要に応じて各ファイルを Read して周辺コンテキストを把握する\n3. 変更行を中心に評価する(変更されていない既存コードの指摘は最小限に留める)\n\n`code-reviewer` の定義にある全観点（8観点）で評価してください（観点はエージェント定義を単一の真実源とし、ここでは列挙しない）。変更箇所に対応するユニットテストの追加・更新も重視してください。\n\nレビューレポートを作成してください。"

### ステップ3: レビュー結果の要約

サブエージェントが作成したレビューレポートの要点を抽出し、ユーザーに報告します。特に優先度が高い問題と、仕様整合性（docs との相違）の指摘を明示します。

## 出力形式

```markdown
# コードレビュー結果

## 対象: [パス または "HEAD との差分（N ファイル）"]

### 主な改善点

1. [改善点1] (優先度: 高/中/低)
2. [改善点2] (優先度: 高/中/低)
3. [改善点3] (優先度: 高/中/低)

### 総合評価

[1-5]/5

### 次のアクション

- [推奨対応1]
- [推奨対応2]

詳細なレポートはサブエージェントの出力を参照してください。
```

## 注意事項

- レビューは詳細な分析のため、数分かかる場合があります
- サブエージェントは独立したコンテキストで動作するため、メインエージェントのコンテキストは消費しません
- 自動修正は行いません。検出された問題の修正は別途実施してください
- 引数なしモードは「ステージ済み + 未ステージ」の差分が対象です。未追跡(untracked)ファイルは含まれません(必要なら `git add` 後に再実行するか、パスを明示してください)
