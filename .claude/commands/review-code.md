# コードレビュー

引数(任意): ファイル/ディレクトリパス(例: `/review-code src/logic/power-index-calc.js`)

引数を省略した場合は **HEAD との差分**(`git diff HEAD`、ステージ済み + 未ステージ)をレビュー対象とする。

## 実行方法

```bash
claude
# パス指定
> /review-code src/logic/power-index-calc.js
# 引数なし: 作業ツリーの未コミット変更をレビュー
> /review-code
```

## 手順

### ステップ1: 対象の決定

- **引数あり**: 指定されたファイル/ディレクトリが存在するか確認する。
- **引数なし**: 以下を実行して差分の対象ファイル一覧と内容を取得する。
  ```bash
  git diff HEAD --name-only   # 変更対象ファイル一覧（ステージ済み + 未ステージ）
  git diff HEAD               # 実際の差分内容
  ```
  対象ファイルが 0 件の場合は「HEAD との差分がありません。レビュー対象がないため終了します」とユーザーに報告して終了する。

### ステップ2: code-reviewerサブエージェント起動

Task tool を使用して code-reviewer サブエージェントを起動する。プロンプトは引数の有無で切り替える。

**引数ありの場合**:
- subagent_type: "code-reviewer"
- description: "Code detailed review"
- prompt: "[対象パス]を詳細にレビューしてください。\n\n以下の観点で評価してください:\n1. 可読性・命名: docs/development-guidelines.md の命名規則・コメント規約・フォーマットに準拠しているか\n2. 設計: 責務分離・純粋関数の原則・関数の長さ・import 依存ルール（src/logic は src/ui や src/data を import しない 等）\n3. テスト: ユニットテストの有無、エッジケースの網羅、命名規則\n4. セキュリティ: 入力検証、innerHTML 禁止、機密情報のハードコード\n5. 仕様整合性: docs/ の永続ドキュメント（PRD・機能設計書・アーキテクチャ）と矛盾していないか\n\nレビューレポートを作成してください。"

**引数なしの場合**(HEAD との差分をレビュー):
- subagent_type: "code-reviewer"
- description: "Code detailed review (HEAD diff)"
- prompt: "HEAD との差分（`git diff HEAD`）を対象にコードレビューを実施してください。\n\n対象ファイル一覧:\n[git diff HEAD --name-only の結果]\n\n手順:\n1. `git diff HEAD` を実行して実際の差分を取得する\n2. 必要に応じて各ファイルを Read して周辺コンテキストを把握する\n3. 変更行を中心に評価する(変更されていない既存コードの指摘は最小限に留める)\n\n以下の観点で評価してください:\n1. 可読性・命名: docs/development-guidelines.md の命名規則・コメント規約・フォーマットに準拠しているか\n2. 設計: 責務分離・純粋関数の原則・関数の長さ・import 依存ルール（src/logic は src/ui や src/data を import しない 等）\n3. テスト: 変更箇所に対応するユニットテストが追加・更新されているか、エッジケースが網羅されているか\n4. セキュリティ: 入力検証、innerHTML 禁止、機密情報のハードコード\n5. 仕様整合性: docs/ の永続ドキュメント（PRD・機能設計書・アーキテクチャ）と矛盾していないか\n\nレビューレポートを作成してください。"

### ステップ3: レビュー結果の要約

サブエージェントが作成したレビューレポートの要点を抽出し、ユーザーに報告します。

## 出力形式

```markdown
# コードレビュー結果

## 対象: [パス または "HEAD との差分（N ファイル）"]

### 主な改善点

1. [改善点1] (優先度: 高/中/低)
2. [改善点2] (優先度: 高/中/低)
3. [改善点3] (優先度: 高/中/低)

### 総合評価

[1-5]/5

### 次のアクション

- [推奨対応1]
- [推奨対応2]

詳細なレポートはサブエージェントの出力を参照してください。
```

## 注意事項

- レビューは詳細な分析のため、数分かかる場合があります
- サブエージェントは独立したコンテキストで動作するため、メインエージェントのコンテキストは消費しません
- 自動修正は行いません。検出された問題の修正は別途実施してください
- 引数なしモードは「ステージ済み + 未ステージ」の差分が対象です。未追跡(untracked)ファイルは含まれません(必要なら `git add` 後に再実行するか、パスを明示してください)
