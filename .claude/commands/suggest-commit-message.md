# コミットメッセージ提案

引数: なし

HEAD との差分(`git diff HEAD`、ステージ済み + 未ステージ)を解析し、
`docs/development-guidelines.md` の Conventional Commits 規約に沿った
コミットメッセージ案を提案する。**コミットは行わない。**

## 実行方法

```bash
claude
> /suggest-commit-message
```

## 手順

### ステップ1: 差分の取得

**このコマンドの開始時に必ず以下を Bash で再実行し、最新の git 状態を取得する。**
セッション中に取得済みの `git status` 出力をキャッシュ的に流用しない(ユーザーが裏で
`git add` / `git commit` / `git restore` を実行している可能性が常にある):

```bash
git status                  # 変更ファイルの全体像（untracked 含む）
git diff HEAD --name-only   # 変更対象ファイル一覧（ステージ済み + 未ステージ）
git diff HEAD               # 実際の差分内容
git log --oneline -10       # 最近のコミットスタイル参考
```

差分が 0 件の場合は「HEAD との差分がありません。コミット対象がないため終了します」と
ユーザーに報告して終了する。**未追跡(untracked)ファイルしか無い場合**は `git diff HEAD` に含まれないため、「`git add` で対象に含めてから再実行してください」と案内して終了する。

差分が著しく大きい場合（目安: 数百ファイル規模・数千行超）は、サブエージェントへ全量を渡さず、ユーザーに「先に論理単位ごとへ分割コミットすることを検討してください」と案内したうえで、可能な範囲で分割案を提示する。

### ステップ2: commit-message-writer サブエージェント起動

Task tool を使用して commit-message-writer サブエージェントを起動する:

- subagent_type: "commit-message-writer"
- description: "Draft commit message"
- prompt: "HEAD との差分を基にコミットメッセージ案を作成してください。\n\n## 差分対象ファイル\n[git diff HEAD --name-only の結果]\n\n## 差分内容\n```diff\n[git diff HEAD の結果]\n```\n\n## 直近コミット履歴（スタイル参考）\n```\n[git log --oneline -10 の結果]\n```\n\n## 制約\n- docs/development-guidelines.md の Conventional Commits 規約に準拠\n- 変更が複数の論理単位にまたがる場合は分割を提案\n- 分割する場合はコミットの適用順序も決定し、依存される変更（前提となる側）を先に並べる。番号順がそのまま git commit 実行順になるようにする\n- コミットは行わず、メッセージ案のみ提示"

### ステップ3: 提案結果の要約

サブエージェントが作成したメッセージ案をユーザーに提示する。**メッセージ全文の提示はここで行う**。変更が複数の論理単位にまたがる場合は、分割案も合わせて提示する。

（メッセージ全文の再掲はここだけでよい。ステップ4 後の最終応答では、保存先パスと `git commit -F` での利用方法の案内に絞る。）

### ステップ4: 推奨メッセージをファイルに保存

サブエージェントが提示したコミットメッセージを `.claude/tmp/commit-msg-N.txt`
(N は 1 から始まる連番)に Write tool で保存する。

#### ステップ4-a: 既存ファイルの一括削除

`.claude/tmp/` は `.gitignore` 済みの一時領域である前提とする（追跡対象に入らない）。書き込みを始める前に、必ず以下を実行して `.claude/tmp/` 配下の既存
`commit-msg-*.txt` をすべて削除する(古い提案の取り残しを防ぐため):

```bash
mkdir -p .claude/tmp
rm -f .claude/tmp/commit-msg-*.txt
```

#### ステップ4-b: 新しいメッセージファイルの書き込み

- 単一コミットの場合は `.claude/tmp/commit-msg-1.txt` のみ作成
- 分割推奨の場合はコミット1 → `commit-msg-1.txt`、コミット2 → `commit-msg-2.txt`、… を作成
  - 番号は**依存関係に基づく適用順**（先にコミットすべきものを `commit-msg-1.txt`）で割り当てる。type の並び順（feat→docs 等）ではなく、依存される変更を先にする
- 「判定根拠」セクションはファイルには含めない(提案メタ情報のため)
- 保存後、ユーザーへの最終応答で以下を明示する:
  - 保存先パス
  - `git commit -F .claude/tmp/commit-msg-1.txt` で直接コミット可能であること

#### ファイル内容のフォーマット

各ファイルは以下の構造で保存する(`git commit -F` でそのままコミットメッセージになる):

```
<type>(<scope>): <subject>

# 概要
- [達成すること1(主たる達成事項)]
  - [達成すること1の補足説明1]
  - [達成すること1の補足説明2]
- [達成すること2]

# 変更内容
- path/to/file1
  - [変更点1]
  - [変更点2]
- path/to/file2
  - [変更点1]
```

- 1行目: 件名(`<type>(<scope>): <subject>`)
- 2行目: 空行
- 3行目以降: `# 概要` セクション(箇条書き)、空行、`# 変更内容` セクション(ファイル別ネスト箇条書き)
- 見出しはレベル 1(`#`)を使用する(`##` 以下は使わない)
- `# 概要` / `# 変更内容` 見出しの直下に空行は入れない(見出し行の次行から箇条書きを始める)
- ファイルパスはバッククォート(`` ` ``)で囲まない(プレーンテキストのまま記載)
- 概要セクションで、主たる達成事項に対する補足説明・付帯情報は **1 段ネストした箇条書き** として記述する(独立した別項目として並列に並べない)
  - 例: 「コマンドを追加」(主) → 「`git commit` は実行しない」「ファイル保存機能あり」(従)
- サブエージェントが本文(必要時のみ)を提示している場合は、件名の直後・`# 概要` の前に空行を挟んで挿入する

## 完了条件

- 推奨メッセージが `.claude/tmp/commit-msg-N.txt`(N は 1 件以上)に保存されている(分割提案時は各コミット分)。
- ユーザーへの最終応答で、保存先パスと `git commit -F .claude/tmp/commit-msg-1.txt` での利用方法が提示されている。
- `git commit` は実行していない。

> 差分が 0 件・untracked のみで終了した場合は、メッセージ案を保存せず案内のみで完了する。

## 出力形式

```markdown
# コミットメッセージ提案

## 対象: HEAD との差分（N ファイル）

## 推奨メッセージ

\`\`\`
<type>(<scope>): <subject>

[必要なら本文]
\`\`\`

### 概要

- [達成すること1]
- [達成すること2]

### 変更内容

- `path/to/file1`
  - [変更点1]
  - [変更点2]
- `path/to/file2`
  - [変更点1]

## 分割推奨（必要時のみ）

複数の論理単位が混在しているため、以下のように分割することを推奨します:

- **コミット1** — [メッセージ案]
  - 概要:
    - [達成すること1]
    - [達成すること2]
  - 変更内容:
    - `path/to/file1`
      - [変更点1]
      - [変更点2]
    - `path/to/file2`
      - [変更点1]
- **コミット2** — [メッセージ案]
  - 概要:
    - [達成すること1]
  - 変更内容:
    - `path/to/file3`
      - [変更点1]
```

## 注意事項

- このコマンドはメッセージ提案のみを行い、`git commit` は実行しません
- untracked ファイルは `git diff HEAD` に含まれないため対象外です（必要なら `git add` 後に再実行）
- サブエージェントは独立したコンテキストで動作するため、メインエージェントのコンテキストは消費しません
- 推奨メッセージは `.claude/tmp/commit-msg-N.txt` に保存されます(`.gitignore` 済み)。`git commit -F .claude/tmp/commit-msg-1.txt` で直接利用可能です
- `.claude/tmp/` は再実行ごとに上書きされる一時領域です
