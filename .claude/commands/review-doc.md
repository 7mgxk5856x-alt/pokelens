# ドキュメントレビュー

引数: レビュー対象のドキュメントパス（必須。例: `/review-doc docs/product-requirements.md`）。本コマンドはパス必須で diff モードはない（ドキュメントは1件ずつレビューする）。

## 実行方法

```bash
claude
> /review-doc docs/product-requirements.md
```

主な対象:

- `docs/product-requirements.md`（PRD）
- `docs/functional-design.md` / `docs/architecture.md` / `docs/repository-structure.md` / `docs/development-guidelines.md` / `docs/glossary.md`

## 手順

### ステップ1: 対象の決定

- **引数あり**: 指定されたドキュメントが存在するか確認する。存在しない場合は「対象が見つかりません。パスを確認してください」とユーザーに報告して終了する。
- **引数なし**: `docs/*.md` を確認して永続ドキュメント一覧を提示し、対象パスの指定を促して終了する。

### ステップ2: doc-reviewerサブエージェント起動

Task tool を使用して doc-reviewer サブエージェントを起動する:

- subagent_type: "doc-reviewer"
- description: "Document detailed review"
- prompt: "[ドキュメントパス]を詳細にレビューしてください。\n\n`doc-reviewer` の定義にある全観点（およびドキュメント種別ごとの追加観点）で評価してください（観点はエージェント定義を単一の真実源とし、ここでは列挙しない）。\n\n必要に応じて `docs/` 配下の他の永続ドキュメントも参照し、一貫性・矛盾を確認してください。\n\nレビューレポートを作成してください。自動修正は行わないでください。"

### ステップ3: レビュー結果の要約

サブエージェントが作成したレビューレポートの要点を抽出し、ユーザーに報告する。特に高優先度の問題と、「完全性の欠落」「他ドキュメントとの不整合」の指摘を明示する。

## 出力形式

```markdown
# ドキュメントレビュー結果

## 対象: [ファイルパス]

### 主な改善点

1. [改善点1] (優先度: 高/中/低)
2. [改善点2] (優先度: 高/中/低)
3. [改善点3] (優先度: 高/中/低)

### 総合評価

[1-5]/5

### 他ドキュメントとの不整合

- [他の docs との矛盾点。なければ「なし」]

### 次のアクション

- [推奨対応1]
- [推奨対応2]

詳細なレポートはサブエージェントの出力を参照してください。
```

## 注意事項

- レビューは詳細な分析のため、数分かかる場合があります
- サブエージェントは独立したコンテキストで動作するため、メインエージェントのコンテキストは消費しません
- 自動修正は行いません。検出された問題の修正は別途実施してください
- 本コマンドはパス必須です（diff モードはありません）。引数を省略した場合は `docs/` 配下の一覧を提示します
