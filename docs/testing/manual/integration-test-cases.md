# 統合テストケース一覧（手動テスト）

pokelens を実際にブラウザで操作して確認する**手動・統合（E2E）テスト**のケース一覧。自動テストがカバーしない領域 ── UI コンポーネントの結合動作・画面表示・データ準備パイプラインの実行確認 ── を、PRD の受け入れ条件に基づいて検証する。

> 自動テストは [../automated/](../automated/)（[単体](../automated/unit-test-cases.md) / [統合](../automated/integration-test-cases.md)）を参照。
>
> **単体テストの手動版は作成しない**: 単体テストはコードレベルの純粋関数・クラス検証であり、Vitest / xUnit が自動実行する。人手で行う相当物が存在しないため。

- **対象バージョン**: Pokémon Champions
- **対象機能**: P0 コア機能（機能1〜6）。P0.5 以降（メガシンカ・ダメージ計算等）は実装後に追記する
- **実行環境**: PC ブラウザ（Chrome 最新版）。`npm run dev` で起動した Vite サーバー経由（`file://` 直接起動は非対応）

> このドキュメントはテストコードではなく PRD の受け入れ条件から手動で起こした仕様書です。機能追加・仕様変更時は併せて更新してください。

## 表の項目

| 項目 | 説明 |
|---|---|
| ケースID | 一意な識別子。`MIT-<連番>` 形式（MIT = Manual Integration Test） |
| 種別 | 正常系 / 異常系 / 境界値 のいずれか |
| テスト観点 | 何を確認したいか |
| 手順 | 実際に行う操作（番号順） |
| 期待結果 | 正しい画面表示・挙動 |
| 備考 | 対応する PRD 機能番号・補足 |

## 事前準備・データ生成

**前提条件**: Node.js（LTS）と .NET SDK 8.0 以上がインストール済み。リポジトリをクローン済み。

| ケースID | 種別 | テスト観点 | 手順 | 期待結果 | 備考 |
|---|---|---|---|---|---|
| MIT-001 | 正常系 | 依存関係のインストール | `npm install` を実行 | エラーなく完了する | セットアップ |
| MIT-002 | 正常系 | マスターデータ生成 | `dotnet run --project tools/PokelensTools` を実行 | `data/` に `pokedex.json` / `moves.json` / `items.json` / `abilities.json` が生成される（威力不定技は moves-power-patch が反映） | 機能6 / データ準備パイプライン |
| MIT-003 | 正常系 | 開発サーバー起動 | `npm run dev` を実行し、表示された URL（既定 `http://127.0.0.1:5173`）をブラウザで開く | アプリ画面が表示される | `file://` 直接起動は非対応 |
