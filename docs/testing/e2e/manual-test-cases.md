# E2E テストケース一覧（手動テスト）

pokelens を実際にブラウザで操作して確認する**手動・E2E（エンドツーエンド）テスト**のケース一覧。自動化できない領域（環境セットアップ・C# データ生成パイプラインの実行確認・dev サーバー起動）に限定する。

> UI 結合・画面表示・サジェスト・火力指数等の機能挙動は自動化済み: [./automated-test-cases.md](./automated-test-cases.md)（Playwright、`npm run test:e2e`）を参照。

> 自動・単体テストは [../unit/automated-test-cases.md](../unit/automated-test-cases.md)、自動・統合テストは [../integration/automated-test-cases.md](../integration/automated-test-cases.md) を参照。

- **対象バージョン**: Pokémon Champions
- **対象機能**: P0 コア機能のセットアップ系（環境構築・データ生成・dev サーバー起動）。それ以外の機能挙動は自動 E2E（Playwright）で担保する
- **実行環境**: PC ブラウザ（Chrome 最新版）。`npm run dev` で起動した Vite サーバー経由（`file://` 直接起動は非対応）
- **優先度**: 本ドキュメントのケースはすべて P0（コア機能の前提）

> このドキュメントはテストコードではなく PRD の受け入れ条件から手動で起こした仕様書です。機能追加・仕様変更時は併せて更新してください。

## 表の項目

| 項目 | 説明 |
|---|---|
| ケースID | 一意な識別子。`MET-<連番>` 形式（MET = Manual E2E Test） |
| 種別 | 正常系 / 異常系 / 境界値 のいずれか |
| 対応要件 | 検証対象の PRD 機能番号（例: `機能2`）。セットアップ作業は `セットアップ` と記す |
| テスト観点 | 何を確認したいか |
| 手順 | 実際に行う操作（番号順） |
| 期待結果 | 正しい画面表示・挙動 |
| 備考 | 補足情報 |

> **採番規則**: ケース ID は末尾追加で連番採番（MET-001 から開始）。MET-004 以降は自動化済みで [./automated-test-cases.md](./automated-test-cases.md) に AET-001〜034 として移行済み。MET-004〜032/034 の番号は欠番扱いとし、再利用しない（自動仕様書との対応関係の追跡性を担保するため）。

## 事前準備・データ生成

**前提条件**: Node.js（LTS）と .NET SDK 8.0 以上がインストール済み。リポジトリをクローン済み。

| ケースID | 種別 | 対応要件 | テスト観点 | 手順 | 期待結果 | 備考 |
|---|---|---|---|---|---|---|
| MET-001 | 正常系 | セットアップ | 依存関係のインストール | `npm install` を実行 | エラーなく完了する | Node.js / npm 環境前提 |
| MET-002 | 正常系 | 機能6 | マスターデータ生成 | `dotnet run --project tools/PokelensMasterDataBuilder` を実行 | `data/` に `pokedex.json` / `moves.json` / `items.json` / `abilities.json` が生成される（威力不定技は moves-power-patch が反映） | データ準備パイプライン。内容の正確性は C# 統合テスト（PIT-003/PIT-007）が担保。本ケースはパイプライン完走とファイル生成の確認のみ |
| MET-003 | 正常系 | セットアップ | 開発サーバー起動 | `npm run dev` を実行し、表示された URL（既定 `http://127.0.0.1:5173`）をブラウザで開く | アプリ画面が表示される | `file://` 直接起動は非対応。自動 E2E は `playwright.config.js` の webServer で内包するため、本ケースは手動セットアップ確認用 |
