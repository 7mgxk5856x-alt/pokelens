# E2E テストケース一覧（手動テスト）

pokelens を実際にブラウザで操作して確認する**手動・E2E（エンドツーエンド）テスト**のケース一覧。自動テストがカバーしない領域 ── UI コンポーネントの結合動作・画面表示・データ準備パイプラインの実行確認 ── を、PRD の受け入れ条件に基づいて検証する。

> 自動テストは [単体](../unit/automated-test-cases.md) / [統合](../integration/automated-test-cases.md) を参照。
>
> **単体テストの手動版は作成しない**: 単体テストはコードレベルの純粋関数・クラス検証であり、Vitest / xUnit が自動実行する。人手で行う相当物が存在しないため。

- **対象バージョン**: Pokémon Champions
- **対象機能**: P0 コア機能（機能1〜6）。P0.5 以降（メガシンカ・ダメージ計算等）は実装後に追記する
- **実行環境**: PC ブラウザ（Chrome 最新版）。`npm run dev` で起動した Vite サーバー経由（`file://` 直接起動は非対応）
- **優先度**: 本ドキュメントのケースはすべて P0（コア機能）。P0.5 以降を追記する際に優先度が分かれる場合は備考に明記する
- **性能**: 非機能要件（起動300ms・サジェスト100ms・切替200ms以内）の検証は手順・期待結果の形式が手動E2Eと異なるため本一覧には含めない（別途計測する）

> このドキュメントはテストコードではなく PRD の受け入れ条件から手動で起こした仕様書です。機能追加・仕様変更時は併せて更新してください。

## 表の項目

| 項目 | 説明 |
|---|---|
| ケースID | 一意な識別子。`MET-<連番>` 形式（MET = Manual E2E Test） |
| 種別 | 正常系 / 異常系 / 境界値 のいずれか |
| 対応要件 | 検証対象の PRD 機能番号（例: `機能2`）。セットアップ作業は `セットアップ` と記す。要件カバレッジの追跡に使う |
| テスト観点 | 何を確認したいか |
| 手順 | 実際に行う操作（番号順） |
| 期待結果 | 正しい画面表示・挙動 |
| 備考 | 補足情報。後始末が必要なケースは「後処理: …」と明記する |

> **採番規則**: ケース ID は末尾追加で連番採番（MET-001 から開始）。サブ採番（例: `MET-025b/c/d`）は同一観点の派生ケース（同種の異常系の入力バリエーション等）に限り使用する。セクション内の物理的並び順は機能別グルーピングを優先するため、ID 番号と物理順序は必ずしも一致しない（番号飛びは許容）。
