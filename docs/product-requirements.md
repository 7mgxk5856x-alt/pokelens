# プロダクト要求定義書 (Product Requirements Document)

## プロダクト概要

### 名称
**pokelens** - ポケモン対戦支援ツール

### プロダクトコンセプト
- **即参照**: 選出画面・対戦中に必要な情報を素早く確認できる
- **火力の見える化**: 自分ポケモンの技ごとに火力指数を事前計算して表示する
- **相手分析**: 相手ポケモンの素早さパターンと基本情報を即座に把握する

### プロダクトビジョン
ポケモン対戦中に「誰を出すべきか」「どの技が通るか」「相手より速いか」を即判断できる軽量ローカルWebツールを提供する。
PCブラウザで起動し、対戦画面と並べて使うことで、情報収集にかかる認知コストを最小化する。
P0では情報参照に特化し、P0.5でメガシンカ対応、P1で対面評価（素早さ判定・ダメージ計算）、P2で対戦状況の動的反映へ段階的に拡張する。P3ではUX・開発体験の向上を図る。

### 目的
- 選出時に必要なデータ（タイプ・特性・種族値・技）を1画面で確認できるようにする
- 自分パーティの火力指数を事前計算し、対戦中に即参照できるようにする
- 相手ポケモンの素早さパターンを表示し、先手・後手の判断を補助する（P0: 6 パターン、P0.5 機能 15 で 4 パターンに簡素化）

### プライマリーペルソナ: 開発者兼プレイヤー（本人）
- ポケモン対戦を楽しむが、種族値・技・特性の暗記に頼りたくない

- 課題: 選出時に誰を出せばいいか判断が遅い、ダメージ感覚がない

- ワークフロー: アプリ起動 → 自パーティ確認 → 相手ポケモン名入力 → 情報参照

## 成功指標（KPI）

個人用ツールのため、主観的な使用感を指標とする。

### プライマリーKPI
- **選出判断時間**: 相手パーティ入力から各ポケモン情報参照まで30秒以内で完了できる
  - 測定方法: 対戦セッション5回分をストップウォッチで計測し、全回30秒以内を達成基準とする
- **継続使用**: リリース後1ヶ月間、対戦セッションの80%以上でツールを開く
  - 測定方法: ブラウザ履歴をもとに対戦セッション数のうちツールを開いたセッション数の割合を週次で自己確認する。80%を達成すれば合格とする

## 機能要件

### P0 コア機能（MVP）

> **P0 実装完了（2026年5月）**。機能 1〜6 は実装・テスト・E2E すべて完了済み。P0.5 以降は未実装。

#### 1. 自分パーティ表示

**ユーザーストーリー**:
対戦プレイヤーとして、自分パーティ6匹の詳細情報を確認するために、アプリ起動時にパーティ一覧が表示されてほしい

**受け入れ条件**:
- [x] 起動時に6匹のポケモンカードが表示される
- [x] 各カードにポケモン名・タイプ・種族値（H-A-B-C-D-S）が表示される
- [x] 起動直後はいずれのカードも未選択状態で、詳細ビューは非表示である
- [x] ポケモンを選択すると詳細ビューに切り替わる
- [x] パーティデータはJSONファイルで定義され、アプリ起動時に読み込まれる
- [x] `party.json` の構文が不正な場合は起動を中断し、エラーメッセージを画面表示する
- [x] 各エントリに必須フィールド（`species`・`nature`・`abilityPoints`・`moves`）が欠落している場合は、構文エラーと同様に起動を中断してエラーメッセージを表示する
- [x] `species` が `pokedex.json` に存在しないポケモンは、該当カードに「不明なポケモン: [species名]」を表示して起動を継続する（他のカードは正常表示される）

**`data/party.json` スキーマ**:

```json
{
  "party": [
    {
      "species": "ガブリアス",
      "ability": "さめはだ",
      "item": "こだわりスカーフ",
      "nature": "いじっぱり",
      "abilityPoints": {
        "hp": 32, "atk": 32, "def": 0,
        "spa": 0, "spd": 0, "spe": 0
      },
      "moves": [
        { "name": "じしん" },
        { "name": "げきりん" },
        { "name": "ストーンエッジ" },
        { "name": "つるぎのまい" }
      ]
    }
  ]
}
```

- `species`: ポケモン名（日本語）。マスターデータとの照合キーとして使用する
- `ability`: 特性名（日本語）。補正の種別（火力・素早さ）と倍率はマスターデータから参照する
- `item`: 持ち物名（日本語）。補正の種別（火力・素早さ）と倍率はマスターデータから参照する
- `nature`: 性格名（日本語）。`data/natures.json` の照合キーとして使用する
- `abilityPoints`: 各ステータスの能力ポイント（0〜32）を `hp` / `atk` / `def` / `spa` / `spd` / `spe` のキーで入力する（各キーの意味は[用語集 - 略語一覧](./glossary.md)を参照）
- `moves`: 最大4つ。技のタイプ・威力・分類・命中率はマスターデータから参照する
  - `name`: 技名（日本語）

**パーティ登録フロー**:

1. ゲーム内のステータス画面で各ポケモンの性格・各ステータスの能力ポイントを確認する
2. テキストエディタで `data/party.json` を開き、スキーマに従って6匹分の情報を入力する
   - `species`: ゲーム内表記のポケモン名を日本語で入力する（例: `"ガブリアス"`）
   - `ability` / `item`: ゲーム内表記をそのまま日本語で入力する
   - `nature`: ゲーム内表記の性格名を日本語で入力する（例: `"いじっぱり"`）
   - `abilityPoints.hp` / `atk` / `def` / `spa` / `spd` / `spe`: 各ステータスの能力ポイント（0〜32）を数値で入力する
   - `moves[].name`: 覚えている技名を日本語で入力する（最大4つ）
3. ファイルを保存し、ブラウザをリロードするとパーティが反映される

> パーティ変更時は `data/party.json` を直接編集してリロードするだけでよい。UIからの編集機能はP0スコープ外。

**優先度**: P0（必須）

#### 2. 自分ポケモン情報表示

**ユーザーストーリー**:
対戦プレイヤーとして、自分ポケモンの詳細を確認するために、ポケモン選択時に実数値・特性・持ち物・技ごとの火力指数を参照したい

**受け入れ条件**:
- [x] 選択した自分ポケモンの「ポケモン名 / タイプ / 特性 / 持ち物 / 性格 / 種族値（H-A-B-C-D-S） / 実数値（H-A-B-C-D-S）」が表示される。実数値は種族値・能力ポイント・性格補正から計算した値を使用する
- [x] 性格表示は性格名に加えて上昇/下降ステータスを `↑`/`↓` で表記する（例: `いじっぱり (A↑ / C↓)`）。補正なし性格は `(補正なし)` と表記する
- [x] 種族値と実数値はそれぞれ `種族値:` / `実数値:` の接頭ラベルを付けて区別する
- [x] 技一覧が表示され、各技に「技名 / タイプ / 威力 / 分類（物理・特殊・変化）/ 命中率 / 火力指数」が並ぶ
- [x] 変化技は火力指数の代わりに「−」を表示する
- [x] 威力不定技（`power: null`）は威力欄を「−」と表示する
- [x] 必中技（`accuracy: null`）は命中率欄を「−」と表示する
- [x] 威力不定技の火力指数は、最大威力パッチファイルに定義がある場合はその値をもとに計算した値を表示し、定義がない技は「−」を表示する（機能6参照）
- [x] 複数回攻撃技の火力指数は、最大総威力（`basePower × multihit[1]`）をもとに計算した値を表示する（機能6参照）
- [x] タイプ名の日本語変換は `data/types.json`（キー: Showdown英語ID、値: 日本語名）から参照する。このファイルは手書きで管理する
- [x] 技分類の日本語変換は `data/move-categories.json`（キー: `"Physical"` / `"Special"` / `"Status"`、値: `"物理"` / `"特殊"` / `"変化"`）から参照する。このファイルは手書きで管理する
- [x] 性格名と能力補正倍率のマッピングは `data/natures.json`（キー: 日本語性格名、値: 各ステータスの補正倍率）から参照する。このファイルは手書きで管理する

> ※ P0.5 で以下の拡張を予定。本受け入れ条件は P0 完了時点の記録として残す。
> - 機能 16: 持ち物が「こだわりスカーフ」の場合、素早さ実数値の横に補正後値を併記
> - 機能 17: 物理耐久指数・特殊耐久指数の追加表示（種族値・実数値それぞれの右隣）

**優先度**: P0（必須）

#### 3. 相手パーティ入力

**ユーザーストーリー**:
対戦プレイヤーとして、相手パーティを素早く入力するために、ポケモン名のサジェスト検索を使って6匹を登録したい

**受け入れ条件**:
- [x] 6つの入力欄に相手ポケモン名を手入力できる
- [x] サジェスト非表示中のタブキーは次の入力欄へフォーカスを移動する
- [x] 入力中にポケモン名候補がサジェスト表示される（前方一致）
- [x] サジェスト候補は最大10件を図鑑番号順で表示する
- [x] サジェスト表示中のタブキー / 下矢印キーは次の候補へフォーカスを移動し、Shift+タブ / 上矢印キーで前の候補へ移動する
- [x] サジェスト表示中は入力欄間のフォーカス移動（タブキーによる）は行わない
- [x] サジェスト表示中にエンターキーで選択中の候補を確定するとそのポケモン名が入力される
- [x] ひらがな・全角カタカナ・半角カタカナ・半角英字（ローマ字）のいずれで入力しても検索できる（半角→全角カタカナ、ひらがな→カタカナ、ローマ字→カタカナに正規化して検索する。ローマ字は訓令式/ヘボン式の両方に対応し、`gabu` → `ガブ` のように母音が確定した分まで貪欲に変換する）
- [x] 半角ハイフン `-` は長音記号 `ー` として扱う（例: `ri-` → `リー` に正規化され、リーフィア等にヒットする）。ローマ字入力時に長音記号をキーボードから直接入力できない補完のため
- [x] 未入力欄は空欄のまま表示される（6匹全員入力は必須でない）
- [x] ポケモン名が存在しない場合は「見つかりません」を表示する
- [x] ポケモン名を確定すると入力欄が非表示になり、自分パーティと同形のカード上に名前・タイプ・種族値（H-A-B-C-D-S）が表示される
- [x] 確定済みカードの右上に「×」ボタンが表示され、クリックするとポケモン名がクリアされ、「×」ボタンが非表示になり再入力可能な状態に戻る。クリアしたスロットが選択中だった場合は相手ポケモン詳細パネルも非表示になる
- [x] 「×」ボタンは Tab フォーカス対象外とする（Tab で 6 つの入力欄を順に移動する際に、確定済みスロットの × でフォーカスが止まらない）

**優先度**: P0（必須）

#### 4. 相手ポケモン情報表示

**ユーザーストーリー**:
対戦プレイヤーとして、相手ポケモンの詳細を確認するために、ポケモン選択時に種族値・特性候補・素早さ6パターンを参照したい

**受け入れ条件**:
- [x] 選択した相手ポケモンの「ポケモン名 / タイプ / 特性候補 / 種族値（H-A-B-C-D-S）」が表示される
- [x] 素早さが6パターン表示される: 最速スカーフ（性格補正1.1×能力ポイント32×持ち物補正1.5）/ 準速スカーフ（補正なし×能力ポイント32×持ち物補正1.5）/ 最速（性格補正1.1×能力ポイント32）/ 準速（補正なし×能力ポイント32）/ 無振り（補正なし×能力ポイント0）/ 最遅（性格補正0.9×能力ポイント0）
- [x] 各パターンは「パターン名: 実数値」の形式で縦に並べて表示する

> ※ P0.5 で以下の拡張を予定。本受け入れ条件は P0 完了時点の記録として残す。
> - 機能 15: 素早さ表示を 4 パターン（最速 / 準速 / 無振り / 最遅）に簡素化
> - 機能 17: 物理耐久指数・特殊耐久指数の 4 パターン表（耐久特化 / 耐久極振 / H極振 / 無振り）の追加表示

**優先度**: P0（必須）

#### 5. 火力指数の計算・表示

**ユーザーストーリー**:
対戦プレイヤーとして、どの技が強いか判断するために、自分ポケモンの技ごとに火力指数が表示されてほしい

**受け入れ条件**:
- [x] 物理技の火力指数: `威力 × 攻撃実数値 × タイプ一致補正 × 特性補正 × 持ち物補正`
- [x] 特殊技の火力指数: `威力 × 特攻実数値 × タイプ一致補正 × 特性補正 × 持ち物補正`
- [x] 変化技は「−」表示（計算しない）
- [x] 威力不定技（`power: null`）は最大威力パッチファイルで定義した値を使用する（機能6参照）。定義がない技は「−」表示とする
- [x] 複数回攻撃技は `basePower × multihit[1]`（最大総威力）を使用する（機能6参照）
- [x] 特性・持ち物の補正はマスターデータから参照する。火力補正が定義されていない場合（補正なし、または素早さ補正のみの場合）は補正値1.0として計算する
- [x] 条件付き補正（技タイプ一致・パンチ技・波動技など）は、その条件を満たす技にのみ補正を適用する。条件を満たさない技には補正値1.0として計算する
- [x] 技タイプ変換系の特性（スキン系: ピクシレート・フリーズスキン・スカイスキン・エレキスキン / ノーマライズ）は、対象タイプの技のタイプを変換した上で STAB 計算に反映する（変換種別は `convertNormalTo`: ノーマル技のみ対象 / `convertAllTo`: 全技対象）

> 対戦状況依存補正（HP・天候・状態異常等）はP0スコープ外（P2 機能 10 で対応）

**優先度**: P0（必須）

---

#### 6. 威力不定技・複数回攻撃技の最大威力計算

**ユーザーストーリー**:
対戦プレイヤーとして、威力不定技や複数回攻撃技の火力感覚も把握するために、より正確な最大威力で火力指数を確認したい

**受け入れ条件**:
- [x] 複数回攻撃技: Pokémon Showdownの `moves.json` で `multihit` フィールドが存在する技は `basePower × multihit[1]`（最大ヒット数）を最大総威力として使用する
- [x] 威力不定技（`power: null`）: `tools/PokelensTools/moves-power-patch.json` に最大威力を手動定義し、C#ツールがShowdownデータのマージ後に上書き適用して `moves.json` へ出力する
- [x] `moves-power-patch.json` は `champions-patch.json` とは独立した別ファイルとして管理する
- [x] パッチファイルに定義されていない威力不定技は「−」表示とする

**優先度**: P0（必須）

### P0.5 拡張機能

#### 7. メガシンカ対応

**ユーザーストーリー**:
対戦プレイヤーとして、メガシンカしたポケモンの情報を正確に参照するために、メガシンカ前後の種族値・タイプ・特性・火力指数を切り替えて確認したい

**受け入れ条件**:

状態定義と切替操作:
- [x] ポケモンの状態は「通常」と「メガシンカ」の 2 種類で表現する
- [x] 起動時およびページリロード時、すべてのポケモンは「通常」状態で表示する（状態は永続化せず、リロードで初期化される）
- [x] メガシンカ可能なポケモンのカードにのみ、状態切替ボタンを表示する（自分パーティ・相手パーティの両方）
- [x] ボタン押下で「通常 → メガシンカ → 通常 → ...」の順に状態が循環する
- [x] メガシンカ先が複数あるポケモンの循環内容は自分側・相手側で異なる:
  - 相手側: 持ち物未知のため全メガ形態を循環する（例: リザードン → メガリザードンＸ → メガリザードンＹ → リザードン → ...）
  - 自分側: 持ち物に対応するメガ形態のみ循環する。リザードン + リザードナイトＸ なら「通常 ↔ メガリザードンＸ」、リザードン + リザードナイトＹ なら「通常 ↔ メガリザードンＹ」
- [x] 切替時、カード表示（ポケモン名・タイプ・種族値）が新しい状態の値に更新される

詳細パネルへの連動:
- [x] ボタン押下したカードが選択状態の場合、表示中のポケモン詳細パネルも連動して新しい状態の値に更新される
  - 自分ポケモン詳細: 名前・タイプ・特性・種族値・実数値（持ち物が「こだわりスカーフ」の場合は素早さ実数値の横にスカーフ補正後値を併記、機能 16）・物理耐久指数・特殊耐久指数（機能 17）・技ごとの火力指数を再計算
  - 相手ポケモン詳細: 名前・タイプ・特性候補・種族値・素早さ 4 パターン（機能 15 で 4 パターンに簡素化）・耐久指数 4 パターン（機能 17）を再計算

メガシンカ発動条件と切替ボタン表示:
- [ ] 自分ポケモンの切替ボタンは、以下のいずれかを満たす場合に表示する:
  - 持ち物がそのポケモンに対応するメガシンカ用アイテム（メガストーン）の場合
    - 例: フシギバナ + フシギバナイト → ボタン表示
    - 例: フシギバナ + カメックスナイト → ボタン非表示（対応アイテム不一致）
    - 例: フシギバナ + こだわりハチマキ → ボタン非表示（メガシンカ用アイテムではない）
  - メガフォームが対応メガシンカ用アイテムを必要としない（メガストーン不要）場合は、持ち物に関わらず常時ボタン表示する
    - 例: レックウザ → 持ち物に関わらず常時ボタン表示（メガレックウザは実ゲーム仕様上ガリョウテンセイ習得が条件だが、本ツールでは技習得チェックは行わない簡略仕様）
- [ ] 相手ポケモンの切替ボタンは持ち物に依存せず、メガシンカ可能なポケモンであれば常に表示する（相手の持ち物は未知のため。メガレックウザもメガシンカ可能ポケモンとして常時表示対象に含む）

データ・計算への反映:

  - 「いいえ」またはメッセージウィンドウ右上の「×」: メッセージウィンドウを閉じ、ツールは閉じない

#### 17. 耐久指数の表示

> 機能 2（自分ポケモン情報表示）・機能 4（相手ポケモン情報表示）の機能追加。

**ユーザーストーリー**:
対戦プレイヤーとして、自分・相手ポケモンの耐久力を即座に把握するために、HP × 防御 / HP × 特防 で計算した耐久指数を表示してほしい

**受け入れ条件**:

計算式:
- [x] 物理耐久指数 = HP実数値 × 防御実数値
- [x] 特殊耐久指数 = HP実数値 × 特防実数値

自分ポケモン詳細:
- [x] 自分ポケモン詳細に、実数値を基に計算した物理耐久指数・特殊耐久指数を表示する
- [x] レイアウト前提として、種族値行（`種族値: H-A-B-C-D-S`）と実数値行（`実数値: H-A-B-C-D-S`）は縦に並べて表示する
- [x] 物理耐久指数は種族値行の右隣、特殊耐久指数は実数値行の右隣に配置する
- [x] 種族値行・実数値行の横幅に関わらず、物理耐久指数欄と特殊耐久指数欄の開始位置（左端）が縦に揃うこと

相手ポケモン詳細:
- [x] 相手ポケモン詳細に、種族値・性格補正・能力ポイントを基に計算した 4 パターンの物理耐久指数・特殊耐久指数を表示する
- [x] 4 パターンの定義（物理耐久指数 / 特殊耐久指数で振り分けと性格補正が異なる）:

  | パターン | 物理耐久指数の前提 | 特殊耐久指数の前提 |
  |---|---|---|
  | 耐久特化 | H32 / B32、性格補正 防御↑(×1.1) | H32 / D32、性格補正 特防↑(×1.1) |
  | 耐久極振 | H0 / B32、性格補正 防御↑(×1.1) | H0 / D32、性格補正 特防↑(×1.1) |
  | H極振 | H32 / B0、性格補正なし | H32 / D0、性格補正なし |
  | 無振り | H0 / B0、性格補正なし | H0 / D0、性格補正なし |

  ※ HP は性格補正の対象外（HP 実数値は性格補正を含まない計算で求める）

- [x] 4 パターンは表形式で横並び表示する
  - 表のヘッダ（列）: 左から「耐久特化」「耐久極振」「H極振」「無振り」の順
  - 行ヘッダ: 上から「物理耐久指数」「特殊耐久指数」の順
  - データセル: ヘッダに対応する数値を表示

**優先度**: P0.5

- ローカル動作のみ。外部への通信は不要（データ取得フェーズを除く）
