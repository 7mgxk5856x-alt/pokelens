# プロダクト要求定義書 (Product Requirements Document)

## プロダクト概要

### 名称
**pokelens** - ポケモン対戦支援ツール

### プロダクトコンセプト
- **即参照**: 選出画面・対戦中に必要な情報を素早く確認できる
- **火力の見える化**: 自分ポケモンの技ごとに火力指数を事前計算して表示する
- **相手分析**: 相手ポケモンの素早さパターンと基本情報を即座に把握する

### プロダクトビジョン
ポケモン対戦中に「誰を出すべきか」「どの技が通るか」「相手より速いか」を即判断できる軽量ローカルWebツールを提供する。
PCブラウザで起動し、対戦画面と並べて使うことで、情報収集にかかる認知コストを最小化する。
P0では情報参照に特化し、P0.5でメガシンカ対応、P1で対面評価（素早さ判定・ダメージ計算）、P2で対戦状況の動的反映へ段階的に拡張する。P3ではUX・開発体験の向上を図る。

### 目的
- 選出時に必要なデータ（タイプ・特性・種族値・技）を1画面で確認できるようにする
- 自分パーティの火力指数を事前計算し、対戦中に即参照できるようにする
- 相手ポケモンの素早さ6パターンを表示し、先手・後手の判断を補助する

## ターゲットユーザー

### プライマリーペルソナ: 開発者兼プレイヤー（本人）
- ポケモン対戦を楽しむが、種族値・技・特性の暗記に頼りたくない
- PCブラウザで対戦しながら別タブで情報参照するスタイル
- 課題: 選出時に誰を出せばいいか判断が遅い、ダメージ感覚がない
- 期待: 対戦中に手元でサクッと確認できるツールがあれば判断が速くなる
- ワークフロー: アプリ起動 → 自パーティ確認 → 相手ポケモン名入力 → 情報参照

## 成功指標（KPI）

個人用ツールのため、主観的な使用感を指標とする。

### プライマリーKPI
- **選出判断時間**: 相手パーティ入力から各ポケモン情報参照まで30秒以内で完了できる
  - 測定方法: 対戦セッション5回分をストップウォッチで計測し、全回30秒以内を達成基準とする
- **継続使用**: リリース後1ヶ月間、対戦セッションの80%以上でツールを開く
  - 測定方法: ブラウザ履歴をもとに対戦セッション数のうちツールを開いたセッション数の割合を週次で自己確認する。80%を達成すれば合格とする

## 機能要件

### P0 コア機能（MVP）

#### 1. 自分パーティ表示

**ユーザーストーリー**:
対戦プレイヤーとして、自分パーティ6匹の詳細情報を確認するために、アプリ起動時にパーティ一覧が表示されてほしい

**受け入れ条件**:
- [ ] 起動時に6匹のポケモンカードが表示される
- [ ] 各カードにポケモン名・タイプが表示される
- [ ] 起動直後はいずれのカードも未選択状態で、詳細ビューは非表示である
- [ ] ポケモンを選択すると詳細ビューに切り替わる
- [ ] パーティデータはJSONファイルで定義され、アプリ起動時に読み込まれる
- [ ] `party.json` の構文が不正な場合は起動を中断し、エラーメッセージを画面表示する
- [ ] 各エントリに必須フィールド（`species`・`nature`・`abilityPoints`・`moves`）が欠落している場合は、構文エラーと同様に起動を中断してエラーメッセージを表示する
- [ ] `species` が `pokedex.json` に存在しないポケモンは、該当カードに「不明なポケモン: [species名]」を表示して起動を継続する（他のカードは正常表示される）

**`data/party.json` スキーマ**:

```json
{
  "party": [
    {
      "species": "ガブリアス",
      "ability": "さめはだ",
      "item": "こだわりスカーフ",
      "nature": "いじっぱり",
      "abilityPoints": {
        "hp": 32, "atk": 32, "def": 0,
        "spa": 0, "spd": 0, "spe": 0
      },
      "moves": [
        { "name": "じしん" },
        { "name": "げきりん" },
        { "name": "ストーンエッジ" },
        { "name": "つるぎのまい" }
      ]
    }
  ]
}
```

- `species`: ポケモン名（日本語）。マスターデータとの照合キーとして使用する
- `ability`: 特性名（日本語）。補正の種別（火力・素早さ）と倍率はマスターデータから参照する
- `item`: 持ち物名（日本語）。補正の種別（火力・素早さ）と倍率はマスターデータから参照する
- `nature`: 性格名（日本語）。`data/natures.json` の照合キーとして使用する
- `abilityPoints`: 各ステータスの能力ポイント（0〜32）を `hp` / `atk` / `def` / `spa` / `spd` / `spe` のキーで入力する（各キーの意味は[用語集 - 略語一覧](./glossary.md)を参照）
- `moves`: 最大4つ。技のタイプ・威力・分類・命中率はマスターデータから参照する
  - `name`: 技名（日本語）

**パーティ登録フロー**:

1. ゲーム内のステータス画面で各ポケモンの性格・各ステータスの能力ポイントを確認する
2. テキストエディタで `data/party.json` を開き、スキーマに従って6匹分の情報を入力する
   - `species`: ゲーム内表記のポケモン名を日本語で入力する（例: `"ガブリアス"`）
   - `ability` / `item`: ゲーム内表記をそのまま日本語で入力する
   - `nature`: ゲーム内表記の性格名を日本語で入力する（例: `"いじっぱり"`）
   - `abilityPoints.hp` / `atk` / `def` / `spa` / `spd` / `spe`: 各ステータスの能力ポイント（0〜32）を数値で入力する
   - `moves[].name`: 覚えている技名を日本語で入力する（最大4つ）
3. ファイルを保存し、ブラウザをリロードするとパーティが反映される

> パーティ変更時は `data/party.json` を直接編集してリロードするだけでよい。UIからの編集機能はP0スコープ外。

**優先度**: P0（必須）

---

#### 2. 自分ポケモン情報表示

**ユーザーストーリー**:
対戦プレイヤーとして、自分ポケモンの詳細を確認するために、ポケモン選択時に実数値・特性・持ち物・技ごとの火力指数を参照したい

**受け入れ条件**:
- [ ] 選択した自分ポケモンの「ポケモン名 / タイプ / 特性 / 持ち物 / 実数値（H-A-B-C-D-S）」が表示される。実数値は種族値・能力ポイント・性格補正から計算した値を使用する
- [ ] 技一覧が表示され、各技に「技名 / タイプ / 威力 / 分類（物理・特殊・変化）/ 命中率 / 火力指数」が並ぶ
- [ ] 変化技は火力指数の代わりに「−」を表示する
- [ ] 威力不定技（`power: null`）は威力欄を「−」と表示する
- [ ] 必中技（`accuracy: null`）は命中率欄を「−」と表示する
- [ ] 威力不定技の火力指数は、最大威力パッチファイルに定義がある場合はその値をもとに計算した値を表示し、定義がない技は「−」を表示する（機能6参照）
- [ ] 複数回攻撃技の火力指数は、最大総威力（`basePower × multihit[1]`）をもとに計算した値を表示する（機能6参照）
- [ ] タイプ名の日本語変換は `data/types.json`（キー: Showdown英語ID、値: 日本語名）から参照する。このファイルは手書きで管理する
- [ ] 技分類の日本語変換は `data/move-categories.json`（キー: `"Physical"` / `"Special"` / `"Status"`、値: `"物理"` / `"特殊"` / `"変化"`）から参照する。このファイルは手書きで管理する
- [ ] 性格名と能力補正倍率のマッピングは `data/natures.json`（キー: 日本語性格名、値: 各ステータスの補正倍率）から参照する。このファイルは手書きで管理する

**優先度**: P0（必須）

#### 3. 相手パーティ入力

**ユーザーストーリー**:
対戦プレイヤーとして、相手パーティを素早く入力するために、ポケモン名のサジェスト検索を使って6匹を登録したい

**受け入れ条件**:
- [ ] 6つの入力欄に相手ポケモン名を手入力できる
- [ ] サジェスト非表示中のタブキーは次の入力欄へフォーカスを移動する
- [ ] 入力中にポケモン名候補がサジェスト表示される（前方一致）
- [ ] サジェスト候補は最大10件を図鑑番号順で表示する
- [ ] サジェスト表示中のタブキーは次の候補へフォーカスを移動し、Shift+タブで前の候補へ移動する
- [ ] サジェスト表示中は入力欄間のフォーカス移動（タブキーによる）は行わない
- [ ] サジェスト表示中にエンターキーで選択中の候補を確定するとそのポケモン名が入力される
- [ ] ひらがな・全角カタカナ・半角カタカナのいずれで入力しても検索できる（半角→全角カタカナ、ひらがな→カタカナに正規化して検索する）
- [ ] 未入力欄は空欄のまま表示される（6匹全員入力は必須でない）
- [ ] ポケモン名が存在しない場合は「見つかりません」を表示する

**優先度**: P0（必須）

---

#### 4. 相手ポケモン情報表示

**ユーザーストーリー**:
対戦プレイヤーとして、相手ポケモンの詳細を確認するために、ポケモン選択時に種族値・特性候補・素早さ6パターンを参照したい

**受け入れ条件**:
- [ ] 選択した相手ポケモンの「ポケモン名 / タイプ / 特性候補 / 種族値（H-A-B-C-D-S）」が表示される
- [ ] 素早さが6パターン表示される: 最速スカーフ（性格補正1.1×能力ポイント32×持ち物補正1.5）/ 準速スカーフ（補正なし×能力ポイント32×持ち物補正1.5）/ 最速（性格補正1.1×能力ポイント32）/ 準速（補正なし×能力ポイント32）/ 無振り（補正なし×能力ポイント0）/ 最遅（性格補正0.9×能力ポイント0）
- [ ] 各パターンは「パターン名: 実数値」の形式で縦に並べて表示する

**優先度**: P0（必須）

---

#### 5. 火力指数の計算・表示

**ユーザーストーリー**:
対戦プレイヤーとして、どの技が強いか判断するために、自分ポケモンの技ごとに火力指数が表示されてほしい

**受け入れ条件**:
- [ ] 物理技の火力指数: `威力 × 攻撃実数値 × タイプ一致補正 × 特性補正 × 持ち物補正`
- [ ] 特殊技の火力指数: `威力 × 特攻実数値 × タイプ一致補正 × 特性補正 × 持ち物補正`
- [ ] 変化技は「−」表示（計算しない）
- [ ] 威力不定技（`power: null`）は最大威力パッチファイルで定義した値を使用する（機能6参照）。定義がない技は「−」表示とする
- [ ] 複数回攻撃技は `basePower × multihit[1]`（最大総威力）を使用する（機能6参照）
- [ ] 特性・持ち物の補正はマスターデータから参照する。火力補正が定義されていない場合（補正なし、または素早さ補正のみの場合）は補正値1.0として計算する
- [ ] 条件付き補正（技タイプ一致・パンチ技・波動技など）は、その条件を満たす技にのみ補正を適用する。条件を満たさない技には補正値1.0として計算する

**優先度**: P0（必須）

---

#### 6. 威力不定技・複数回攻撃技の最大威力計算

**ユーザーストーリー**:
対戦プレイヤーとして、威力不定技や複数回攻撃技の火力感覚も把握するために、より正確な最大威力で火力指数を確認したい

**受け入れ条件**:
- [ ] 複数回攻撃技: Pokémon Showdownの `moves.json` で `multihit` フィールドが存在する技は `basePower × multihit[1]`（最大ヒット数）を最大総威力として使用する
- [ ] 威力不定技（`power: null`）: `tools/PokelensTools/moves-power-patch.json` に最大威力を手動定義し、C#ツールがShowdownデータのマージ後に上書き適用して `moves.json` へ出力する
- [ ] `moves-power-patch.json` は `champions-patch.json` とは独立した別ファイルとして管理する
- [ ] パッチファイルに定義されていない威力不定技は「−」表示とする

**優先度**: P0（必須）

### P0.5 拡張機能

#### 7. メガシンカ対応

**ユーザーストーリー**:
対戦プレイヤーとして、メガシンカしたポケモンの情報を正確に参照するために、メガシンカ前後の種族値・タイプ・特性・火力指数を切り替えて確認したい

**受け入れ条件**:
- [ ] メガシンカ可能なポケモンには切り替えボタンが表示される（自分パーティ・相手ポケモンの両方）
- [ ] 切り替えると、種族値・タイプ・特性がメガシンカ後の値に更新される
- [ ] 自分ポケモンがメガシンカ状態の場合、メガシンカ後の種族値と `party.json` の能力ポイント・性格から実数値を再計算し、火力指数に反映する
- [ ] 相手ポケモンがメガシンカ状態の場合、素早さ4パターンはメガシンカ後の種族値で再計算される
- [ ] メガシンカデータ（変化後の種族値・タイプ・特性）はマスターデータに含まれる

**優先度**: P0.5

### P1 拡張機能

#### 8. 対面選択と素早さ判定

**ユーザーストーリー**:
対戦プレイヤーとして、対面する2匹を選んで先手・後手を確認するために、自分と相手のポケモンを1匹ずつ選択して素早さ判定を見たい

**受け入れ条件**:
- [ ] 自分ポケモン・相手ポケモンをそれぞれ1匹選択できる
- [ ] 相手の素早さ4パターンそれぞれに対して「自分が先手 / 後手 / 同速」を判定して表示する

**優先度**: P1（重要）

---

#### 9. ダメージ割合表示

**ユーザーストーリー**:
対戦プレイヤーとして、技を打ったときのダメージ感覚を把握するために、相手への推定ダメージ割合を確認したい

**受け入れ条件**:
- [ ] 対面選択した組み合わせで、自分の技が相手に与えるダメージの相手HP比（ダメージ ÷ 相手HP実数値 × 100）を表示する
- [ ] 相手の能力ポイントは4パターンを仮定して計算する:
  - HP能力ポイント32 + 防御/特防能力ポイント32（性格補正あり）
  - HP能力ポイント32 + 防御/特防能力ポイント0（補正なし）
  - HP能力ポイント0 + 防御/特防能力ポイント32（補正あり）
  - HP能力ポイント0 + 防御/特防能力ポイント0（補正なし）
- [ ] ダメージ割合は「最小〜最大 %」の範囲で表示する

**優先度**: P1（重要）

---

### P2 将来機能

#### 10. 対戦状況の入力と計算への反映

**ユーザーストーリー**:
対戦プレイヤーとして、実際の対戦状況に合わせた計算結果を確認するために、天候・フィールド・能力変化・現在HPを入力し、火力指数・ダメージ計算に反映させたい

**受け入れ条件**:
- [ ] 天候（晴れ・雨・砂嵐・雪など）を選択でき、該当する技の火力補正に反映される
- [ ] フィールド（エレキ・グラス・ミスト・サイコ）を選択でき、該当する技の火力補正に反映される
- [ ] 自分・相手それぞれの能力変化ランク（各ステータス±6段階）を入力でき、ダメージ計算に反映される
- [ ] 相手ポケモンの現在HPを入力でき、ダメージ割合計算の基準HPとして使用される
- [ ] 対戦状況はポケモン選択中のみ保持し、ポケモンを切り替えるとリセットされる

> 能力変化ランク・現在HP入力の項目はP1機能8（ダメージ割合表示）の実装を前提とする。

**優先度**: P2

### P3 将来機能（UX・開発体験の向上）

#### 11. サジェスト対象の対応バージョン絞り込み

**ユーザーストーリー**:
対戦プレイヤーとして、相手パーティ入力時に対象バージョン外のポケモンが候補に出ないようにするために、サジェスト対象をPokémon Championsで使用可能なポケモンのみに絞り込みたい

**受け入れ条件**:
- [ ] 相手パーティ入力のサジェスト候補が、Pokémon Championsで使用可能なポケモンのみに限定される
- [ ] 使用可能ポケモンリストはマスターデータに含まれ、バージョン更新時はC#ツールの再実行で反映される

**優先度**: P3

---

#### 12. ポケモン・タイプアイコン表示

**ユーザーストーリー**:
対戦プレイヤーとして、ポケモンとタイプを視覚的に素早く認識するために、ポケモン名の横にアイコン画像・タイプを文字ではなくアイコンで表示してほしい

**受け入れ条件**:
- [ ] ポケモン名の横にポケモンのアイコン画像が表示される
- [ ] タイプが文字表記の代わりにタイプアイコン（色付き画像）で表示される

**優先度**: P3

---

#### 13. 優れた種族値の強調表示

**ユーザーストーリー**:
対戦プレイヤーとして、相手ポケモンの強みを一目で把握するために、高い種族値の項目が視覚的に強調表示されてほしい

**受け入れ条件**:
- [ ] 種族値が一定の閾値を超える項目が強調表示される（色・太字など）
- [ ] 強調の閾値・表示方法はUIデザイン時に決定する

**優先度**: P3

---

#### 14. C#ツール — データ編集UI

**ユーザーストーリー**:
開発者として、`champions-patch.json` および `party.json` をGUIで安全に編集するために、C#ツールに編集インターフェースを追加してほしい

**受け入れ条件**:
- [ ] `champions-patch.json` のエントリを追加・編集・削除できるGUIまたはCLIインターフェースが提供される
- [ ] `party.json` のパーティ情報（ポケモン・技・持ち物・実数値）を編集できるインターフェースが提供される
- [ ] 保存時にJSONスキーマのバリデーションを行い、不正な値は保存前にエラーを表示する

**優先度**: P3

---

### スコープ外（全フェーズで実装しない）
- 有利不利の自動判定・交代推奨
- AIによる意思決定支援
- オンライン同期・データ保存

## 非機能要件

### パフォーマンス
- アプリ起動時の JSON 読み込み完了: 300ms以内
- ポケモン名サジェストの表示: 入力から100ms以内
- 相手ポケモン情報の表示切り替え: クリックから200ms以内

> 測定方法は [アーキテクチャ設計書 - パフォーマンス要件](./architecture.md#パフォーマンス要件) を参照。

### ユーザビリティ
- PCブラウザ（Chrome最新版）で動作すること
- ローカル環境で `npm run dev` によりViteサーバーを起動してブラウザで開くこと（`file://` 直接起動は非対応）
- 対戦中にもう一方の手でポケモン名を打ちながら参照できる程度のシンプルなUI

### 信頼性
- 自分パーティデータはJSONファイル1つで管理し、編集・再起動で即反映されること
- Pokémon ShowdownデータはC#ツールで事前取得・変換してJSONとして保持すること（ランタイムで外部APIを叩かない）

### セキュリティ
- ローカル動作のみ。外部への通信は不要（データ取得フェーズを除く）
